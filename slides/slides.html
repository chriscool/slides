Disponible sur https://git-scm.com/

Let's begin!

On crée un répertoire qui accueillera notre dépôt et on initialise un dépôt (repo) Git dedans

mkdir ma-recette cd ma-recette git init

Ou bien, si on préfère (et connaît un peu la programmation)

mkdir mes-scripts cd mes-scripts git init

L'index

On crée un fichier "recette.txt" ou "utils.sh" et on demande a Git de suivre ce fichier

touch NOM-DE-FICHIER-CHOISI-CI-DESSUS git status

git status nous indique l'état de notre dépôt. Ici on voit que notre fichier n'est pas suivi, il faut l'ajouter à l'index (des fichiers suivis par Git).

git add NOM-DE-FICHIER-CHOISI-CI-DESSUS git status

Le commit

Maintenant que notre index est prêt, on peut créer un commit.

git commit git show git status git log

Commiter du contenu dans le fichier

On peut maintenant éditer le fichier, le modifier et y ajouter ou retirer du contenu.

Par exemple si on utilise le programme "nano" comme éditeur de texte

nano NOM-DE-FICHIER-CHOISI git commit NOM-DE-FICHIER-CHOISI git show git log

Le revert

Imaginons qu'une erreur se glisse dans notre commit :

git rm recette.txt && git commit

Heureusement git peut facilement revenir dans le temps !

git revert HEAD

Get remote!

Après ce travail en local, il est temps de partager la recette.

Créez un dépôt sur GitHub avec le nom que vous voulez et ne l'initialisez pas.

git remote add origin https://github.com/$USER/atelier-git.git git push origin master

Vous pouvez maintenant voir votre code sur GitHub !

Fork

Il est possible de partir du travail de quelqu'un d'autre depuis GitHub. Allez sur le dépôt et cliquez sur "Fork"

GitHub crée une copie chez vous. C'est de là que vous pouvez partir :

git clone $ADRESSE_DE_VOTRE_DEPOT

Vous pouvez maintenant modifier comme bon vous semble les fichiers !

Pull Request

Comme il est interdit de pousser du contenu sur un dépôt qui ne nous appartient pas, on passe par une Pull Request

Dès que le propriétaire aura accepté votre requête, votre modification sera intégrée dans son dépôt.
