Pourquoi Git ?

  • Libre
  • Décentralisé
  • Local
  • Rapide
  • Utilisé par au moins 80% des développeurs dans le monde

GitLab et GitHub ne sont pas lié à Git, il s'agit juste de plateformes offrant des dépôts Git gratuits. On trouve aussi BitBucket, SourceForge, Savannah, AWS CodeCommit, etc.

Git est disponible et téléchargeable depuis https://git-scm.com/

Let's begin!

On crée un répertoire qui accueillera notre dépôt et on initialise un dépôt (repo) Git dedans

mkdir ma-recette cd ma-recette git init

Ou bien, si on préfère (et connaît un peu la programmation)

mkdir mes-scripts cd mes-scripts git init

L'index

On crée un fichier "recette.txt" ou "utils.sh" et on demande a Git de suivre ce fichier

touch NOM-DE-FICHIER-CHOISI-CI-DESSUS git status

git status nous indique l'état de notre dépôt. Ici on voit que notre fichier n'est pas suivi, il faut l'ajouter à l'index (des fichiers suivis par Git).

git add NOM-DE-FICHIER-CHOISI-CI-DESSUS git status

Le commit

Maintenant que notre index est prêt, on peut créer un commit.

git commit git show git status git log

Commiter du contenu dans le fichier

On peut maintenant éditer le fichier, le modifier et y ajouter ou retirer du contenu.

Par exemple si on utilise le programme "nano" comme éditeur de texte

nano NOM-DE-FICHIER-CHOISI # ajouter du contenu dans le fichier git commit NOM-DE-FICHIER-CHOISI git show git log

Quel contenu mettre dans le fichier

Si vous éditez une recette de cuisine ("recette.txt") vous pouvez expliquer de quel plat il s'agit, ingrédients nécessaires, la recette proprement dite, etc.

Si vous éditez éditez un script shell ("utils.sh"), vous pouvez vous créer des alias ou des fonctions Shell pour vous simplifier la vie avec Git, par exemple:

alias glog='git log' alias gdog='git log --decorate --oneline --graph' grp() { git grep -l "$1" | xargs perl -pi -e "s/$1/$2/g"; }

Vous pouvez ensuite "sourcer" votre script dans le shell courant (ou dans votre ~/.bashrc) pour pouvoir utiliser les alias et fonctions que vous avez créés:

. utils.sh gdog

Le revert

Imaginons qu'une erreur se glisse dans notre commit :

git rm recette.txt git commit

Heureusement git peut facilement revenir dans le temps !

git revert HEAD

Get remote!

Après ce travail en local, il est temps de partager la recette.

Créez un dépôt sur GitLab ou GitHub avec le nom que vous voulez et ne l'initialisez pas.

git remote add origin https://$GITLAB_OU_GITHUB.com/$USER/atelier-git.git git push origin master

Vous pouvez maintenant voir votre code sur GitLab ou GitHub !

Fork

Il est possible de partir du travail de quelqu'un d'autre depuis GitLab ou GitHub. Allez sur le dépôt et cliquez sur "Fork"

GitHub ou GitLab crée une copie chez vous. C'est de là que vous pouvez partir :

git clone $ADRESSE_DE_VOTRE_DEPOT

Vous pouvez maintenant modifier comme bon vous semble les fichiers !

Merge ou Pull Request

Comme il est interdit de pousser du contenu sur un dépôt qui ne nous appartient pas, on passe par une Pull Request (GitHub) ou une Merge Request (GitLab)

Dès que le propriétaire aura accepté votre requête, votre modification sera intégrée dans son dépôt.
